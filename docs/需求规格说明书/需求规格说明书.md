## 1. 引言

### 1.1 项目背景与目标

#### 项目背景

大学校园环境宜人、绿化率高，常有流浪猫狗在此栖息。这些动物给校园带来生机与活力的同时，也带来了一些管理问题：

- **管理者视角**：哪些区域动物出没频繁？是否需要设置安全提示？如何系统化地记录和管理校园流浪动物信息？目前缺乏一个统一的数据平台支撑管理决策。
- **学生/游客视角**：想去看看校园里的猫狗，但不知道去哪、什么时候去能见到；想了解动物们的动态却只能靠偶遇和口口相传。

校园流浪动物管理长期处于"靠经验、靠运气"的状态——缺少数据采集手段、缺少系统化管理工具、缺少面向公众的信息服务。开发一套集数据采集、智能识别、信息管理与公共服务于一体的系统，成为解决上述问题的可行路径。

#### 项目目标

开发一个**校园流浪动物观测关爱系统**，通过采集校园各区域的动物影像，利用 YOLOv8 深度学习模型自动检测和识别猫狗品种，为两类用户群体提供完整的服务闭环：

| 用户群体 | 核心诉求 | 系统提供的价值 |
|:--------|:--------|:-------------|
| **校园管理者** | 了解动物分布、设置安全提示、管理数据 | 数据看板、检测管理、安全提醒发布、周报月报导出 |
| **学生/游客** | 了解动物动态、获取趣味内容、注意安全 | 实时大屏、出没日历、排行榜、撸猫指南、安全提醒公示 |

核心数据流：

```
摄像头/照片 → YOLOv8 品种检测 → 数据库 → 管理员管理端
                                        → 公共展示端（大屏/日历/排行榜等）
```

系统打通了"采集 → 识别 → 入库 → 分析 → 产出"的完整链路，将检测数据转化为安全提醒、撸猫指南等有价值的输出。

**流程说明：**

1. 管理员登录系统，进入管理端总览看板
2. 选择拍摄地点 → 上传图片 → 系统触发 YOLO 自动检测
3. 查看检测结果（各动物的品种、置信度、位置框），决定保存记录或重新检测
4. 可从检测结果生成分享卡片，供公共端使用
5. 可在安全提醒管理页创建、发布或下架安全提醒
6. 可按需导出周报/月报，或在看板上查看全局数据统计

**result_json 格式说明：**

```json
[
  {
    "breed": "Bengal",
    "confidence": 0.96,
    "box_x": 120,
    "box_y": 340,
    "box_w": 180,
    "box_h": 220
  },
  {
    "breed": "British_Shorthair",
    "confidence": 0.88,
    "box_x": 450,
    "box_y": 280,
    "box_w": 150,
    "box_h": 190
  }
]
```

> 设计说明：用 JSON 字段而非展开成多行表，因为一张图片的检测结果天然是一个整体（一起读写），避免额外建 detection_item 表，查询更简单。SQLite 原生支持 JSON 函数查询聚合。

**表4：safety_tip（安全提醒发布记录）**

| 字段 | 类型 | 空 | 默认值 | 约束 | 说明 |
|:----|:----|:--|:------|:-----|:-----|
| id | INTEGER | NOT NULL | 自增 | PRIMARY KEY | 提醒唯一标识 |
| location_id | INTEGER | NOT NULL | — | FOREIGN KEY → location.id | 关联的地点 |
| title | TEXT | NULL | NULL | — | 提醒标题 |
| content | TEXT | NOT NULL | — | — | 提醒正文内容 |
| status | TEXT | NULL | 'published' | CHECK(status IN ('published','draft','archived')) | 状态：已发布/草稿/已归档 |
| created_at | DATETIME | NULL | CURRENT_TIMESTAMP | — | 创建时间 |

> 注意：`location.safety_tip` 存当前生效的一条提醒文案（方便公共端读取），`safety_tip` 表存完整的历史发布记录。两者配合使用——新建/修改提醒时同时更新两个字段，公共端只查 location 表即可。

### 3.5 安全性需求

- **登录鉴权**：管理员端所有操作需通过密码验证，未登录用户无法访问管理功能
- **密码存储**：用户密码不存明文，使用哈希算法（bcrypt 或 hashlib）存储密码哈希值
- **上传校验**：上传文件做类型校验，仅允许 JPG / PNG 图片格式，拒绝可执行文件等危险类型
- **文件大小限制**：单张上传图片不超过 10MB
- **API 鉴权**：管理端 API 接口通过 Session / Token 机制进行基本鉴权；公共端接口无需鉴权
- **SQL 注入防护**：使用 ORM 或参数化查询，不拼接 SQL 字符串
- **路径遍历防护**：图片存储使用系统生成的唯一文件名（UUID），不直接使用用户提供的文件名
